# 个人记账APP - 产品文档

## 项目概述

本项目是一款**个人记账APP**，帮助用户记录和管理日常收支。

- **货币单位**：人民币（元）
- **核心功能**：记录每一笔收入和支出，并按两级分类进行归类
- **目标用户**：个人日常记账使用

---

## 核心功能需求

1. **记录收支**：选择收入或支出类型，输入金额、选择分类、自动记录日期时间
2. **收支分类**：两级分类体系（一级大类 → 二级小类），支出12类 + 收入7类
3. **备注说明**：每笔记录可添加文字备注
4. **历史记录**：查看过往所有收支记录，支持按月份和分类筛选
5. **统计报表**：分别查看收入/支出分类占比饼图、排行榜和月度结余

### ✅ 方案A：纯前端Web应用 — **已选定**

| 项目 | 说明 |
|------|------|
| **技术** | HTML + CSS + JavaScript，数据存浏览器本地存储（localStorage） |
| **需要什么** | 一个浏览器就能用，不需要任何服务器 |
| **优点** | 开发最简单、零成本、打开网页就能用、不需要网络 |
| **缺点** | 换设备或清浏览器缓存数据会丢失，不能多设备同步 |
| **适合谁** | 只用一台电脑/手机记账、追求简单、快速上手 |

> 📝 **用户决策记录（2026-07-08）**：用户选择了方案A，优先追求简单易用和快速上手。

### 方案B：前端 + 云数据库 ⭐推荐实用

| 项目 | 说明 |
|------|------|
| **技术** | React 或 Vue 框架 + Supabase 或 Firebase 云数据库 |
| **需要什么** | 注册一个免费云服务账号（Supabase/Firebase） |
| **优点** | 数据存放在云端不会丢失、可以多设备登录同步、有现成的注册登录功能 |
| **缺点** | 需要联网使用，免费额度有上限（个人记账完全够用） |
| **适合谁** | 希望在手机和电脑上都能记账、担心数据丢失 |

### 方案C：微信小程序

| 项目 | 说明 |
|------|------|
| **技术** | 微信小程序官方框架 |
| **需要什么** | 微信开发者工具、微信小程序账号 |

## 用户协作规则

> **重要**：以下规则适用于整个开发过程，请AI严格遵守。

1. **用户是技术小白**：AI在沟通时应避免使用晦涩的专业术语，如果必须使用，需要附带通俗解释。
2. **方案由AI列，决策由用户做**：每次面临技术选型或设计决策时，AI需要提供2-4个备选方案，每个方案都列出优点和缺点，由用户来做最终选择。
3. **AI不自行决定方向**：在没有得到用户明确选择之前，AI不得自行选定技术方案或架构设计。
4. **解释优先**：在给出方案时，用"这个方案适合谁"、"需要什么条件"等通俗的角度来解释，而不仅仅是技术参数。
